Service Catalog

Only five standardized request types are supported — SDLC cloud execution for approved workloads (Sandbox / Dev / UAT)

⚠️ Cloud Frontdoor Guardrail

This catalog is only applicable after a workload has received a positive Cloud Frontdoor outcome. Cloud Frontdoor remains the system of record for cloud suitability, onboarding, and risk decisions. This model operationalizes execution once those decisions are made.

🏗️ 6.1 Environment Provisioning Request

Purpose: SDLC environment execution only

Guardrails: Non‑production only (Sandbox / Dev / UAT) · Requires prior Cloud Frontdoor approval (informational reference)

Key Fields

Application Name Business Owner Technical Owner Cost Center Environment Type (Sandbox / Dev / UAT) Region Data Classification Criticality Tier Internet Exposure (Yes / No) RTO / RPO Cloud Frontdoor Reference ID (optional) Expected Go‑Live Date Notes
🏗️ Submit Environment Request

🔑 6.2 Identity & Access Request (Non‑Human)

Purpose: Workload‑level identity enablement

Scope

Managed Identity Service Principal

Key Fields

Application Name Identity Type Scope (Subscription / Resource Group) Role Requested Temporary Access (Yes / No) Duration (if temporary) Business Justification
🔑 Submit IAM Request

🌐 6.3 Network & Connectivity Request

Purpose: Controlled connectivity enablement

Request Types

Private Endpoint Firewall Rule API Exposure

Key Fields

Application Name Source / Destination Protocol / Port Public Exposure (Yes / No) Third‑Party Integration (Yes / No)
🌐 Submit Network Request

⚙️ 6.4 Platform Services Request

Purpose: Standard service provisioning

Supported Services

Database Storage Queue Cache AI / ML Services

Key Fields

Application Name Service Type SKU (Standard / Premium / Custom) Estimated Monthly Cost Backup Required (Yes / No) Monitoring Required (Yes / No)
⚙️ Submit Platform Request

🛡️ 6.5 Security Exception Request

Purpose: Temporary deviation from enforced policies

Rule: No permanent security exceptions allowed. Expiry date is mandatory.

Key Fields

Policy / Control Violated Business Justification Compensating Controls Expiry Date (Mandatory) Risk Acknowledgement
🛡️ Submit Security Exception

☁️ 6.6 New Subscription Creation Request

Purpose: Provision a new cloud subscription for an approved workload

Scope: Non‑production subscriptions only (Sandbox / Dev / UAT) · Requires prior Cloud Frontdoor approval

Key Fields

Subscription Name Business Owner Technical Owner Cost Center Subscription Purpose (Sandbox / Dev / UAT) Management Group Budget Limit (monthly) Region Data Classification Business Justification Cloud Frontdoor Reference ID (optional) Notes
☁️ Submit Subscription Request

📋 Scope Boundary

✅ In Scope

  • SDLC cloud execution requests (Sandbox, Dev, UAT)
  • Non‑human IAM at workload level
  • Network connectivity enablement
  • Standard platform services provisioning
  • Time‑bound security exceptions (execution only)

❌ Explicitly Out of Scope

  • Cloud suitability decisions (Cloud Frontdoor)
  • Experimentation vs Production pathway
  • Production environment provisioning
  • User IAM and break‑glass access
  • Incident and change management
  • Regulatory risk acceptance decisions
